
# 常见问题与回答

1. 提示`libtbe_adapter.so does not exist`  
   您可通过安装nnal软件包解决该问题，`ATB_BUILD_DEPENDENCY_PATH`设置方式见下：
    - 安装步骤可参考[run包使用](#run-package-usage)
    - 代码及软件包版本对应关系：  
        nnal软件包需保持和toolkit及kernels软件包版本一致
    - 设置`ATB_BUILD_DEPENDENCY_PATH`（以下二选一）：
        - 不设置：仅当 nnal 安装在默认位置（`/usr/local/Ascend/nnal/...`，即 root 默认安装路径）时适用；此时脚本会按检测到的 ABI 自动使用默认路径`/usr/local/Ascend/nnal/atb/latest/atb/cxx_abi_0`或`cxx_abi_1`，ABI 与编译保持一致，无需手动指定。
        - 手动设置（nnal 安装在其他位置时）：须保证`cxx_abi_{cxx_abi_version}`与编译实际使用的 ABI 一致。

            ```sh
            export ATB_BUILD_DEPENDENCY_PATH={nnal install path}/nnal/atb/latest/atb/cxx_abi_{cxx_abi_version}
            ```

            `{cxx_abi_version}`的确定方法：先判断是否安装 PyTorch——下面命令报错（如`ModuleNotFoundError: No module named 'torch'`）即未安装，正常打印版本号即已安装：

            ```sh
            python3 -c "import torch; print(torch.__version__)"
            ```

            - 未安装 PyTorch：使用`cxx_abi_1`
            - 已安装 PyTorch：运行下面命令，输出`1`用`cxx_abi_1`，输出`0`用`cxx_abi_0`

                ```sh
                python3 -c "import torch; print(1 if torch.compiled_with_cxx11_abi() else 0)"
                ```

            示例（nnal 安装在`/home/user/Ascend/nnal`）：

            - 未安装 PyTorch：`export ATB_BUILD_DEPENDENCY_PATH=/home/user/Ascend/nnal/atb/latest/atb/cxx_abi_1`
            - 已安装 PyTorch 且输出`0`：`export ATB_BUILD_DEPENDENCY_PATH=/home/user/Ascend/nnal/atb/latest/atb/cxx_abi_0`
    - run包使用<a id="run-package-usage"></a>
       - run包获取
          1. 进入网址：<https://www.hiascend.com/developer/download/community>
          2. 产品系列选择服务器，产品型号根据设备型号选择，选择所需解决方案版本，随后在CANN区域选择软件包跟随指引即可获取相关run包
       - 软件包名为：Ascend-cann-nnal_{version}_linux-{arch}.run  
       其中，{version}表示软件版本号，{arch}表示CPU架构。
       - 安装run包（需要依赖cann环境）

           ```sh
           chmod +x 软件包名.run # 增加对软件包的可执行权限
           ./软件包名.run --check # 校验软件包安装文件的一致性和完整性
           ./软件包名.run --install # 安装软件，可使用--help查询相关安装选项
           ```

           出现提示`xxx install success!`则安装成功
